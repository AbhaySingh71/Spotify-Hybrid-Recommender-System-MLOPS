POTIFY HYBRID RECOMMENDER SYSTEM - COMPLETE AWS MLOPS DEPLOYMENT GUIDE

PHASE 1: DATA VERSIONING USING AWS S3 + DVC

1.  Create S3 Bucket

-   Go to AWS Console → S3
-   Create bucket (example: spotify-hybrid-recsys-data)
-   Disable public access (recommended)
-   Enable versioning (recommended)

2.  Create IAM User

-   Go to IAM → Users → Create User
-   Attach Policy: AmazonS3FullAccess (for learning)
-   Generate Access Key ID and Secret Access Key

3.  Configure AWS CLI Run: aws configure

Enter: - AWS Access Key - AWS Secret Key - Region (e.g., us-east-1) -
Output format: json

4.  Add DVC Remote dvc remote add -d myremote
    s3://spotify-hybrid-recsys-data dvc remote modify myremote
    access_key_id YOUR_ACCESS_KEY dvc remote modify myremote
    secret_access_key YOUR_SECRET_KEY

Push data: dvc push

PHASE 2: DOCKERIZATION

1.  Create Dockerfile

FROM python:3.11-slim WORKDIR /app COPY requirements.txt . RUN pip
install –no-cache-dir -r requirements.txt COPY . . EXPOSE 8000 CMD
[“python”, “app.py”]

2.  Build Docker Image docker build -t spotify_hybrid_resys:latest .

3.  Test Locally docker run -d –name hybrid_recsys -p 8000:8000
    spotify_hybrid_resys:latest

Open: http://localhost:8000

PHASE 3: AWS ECR SETUP

1.  Create ECR Repository

-   Go to AWS → ECR → Create Repository
-   Name: spotify-hybrid-recsys

2.  Authenticate Docker to ECR aws ecr get-login-password –region
    us-east-1 | docker login –username AWS –password-stdin
    .dkr.ecr.us-east-1.amazonaws.com

3.  Tag Image docker tag spotify_hybrid_resys:latest
    .dkr.ecr.us-east-1.amazonaws.com/spotify-hybrid-recsys:latest

4.  Push Image docker push
    .dkr.ecr.us-east-1.amazonaws.com/spotify-hybrid-recsys:latest

5.  Test Pull docker pull
    .dkr.ecr.us-east-1.amazonaws.com/spotify-hybrid-recsys:latest docker
    run -d –name hybrid_recsys -p 8000:8000
    .dkr.ecr.us-east-1.amazonaws.com/spotify-hybrid-recsys:latest

PHASE 4: CI/CD PIPELINE (GitHub Actions)

Pipeline Steps: 1. Checkout Code 2. Setup Python 3. Install Dependencies
4. Run Tests 5. Build Docker Image 6. Login to ECR 7. Push to ECR 8.
Upload artifacts to S3 for CodeDeploy

PHASE 5: EC2 + AUTO SCALING + LOAD BALANCER

1.  Create IAM Role for EC2 Attach Policies:

-   AmazonS3FullAccess
-   AmazonEC2ContainerRegistryFullAccess
-   AmazonEC2FullAccess
-   AWSCodeDeployFullAccess

2.  Create Launch Template

-   Select Ubuntu
-   Enable Auto Scaling guidance
-   Add Key Pair
-   Allow all traffic (for study only)
-   Attach IAM Role

3.  Add User Data Script

#!/bin/bash sudo apt update -y sudo apt install ruby-full -y sudo apt
install wget -y cd /home/ubuntu wget
https://aws-codedeploy-us-east-1.s3.us-east-1.amazonaws.com/latest/install
chmod +x install sudo ./install auto sudo systemctl start
codedeploy-agent sudo systemctl enable codedeploy-agent

4.  Create Auto Scaling Group

-   Select Launch Template
-   Select Availability Zones
-   Attach Load Balancer
-   Enable Health Checks
-   Create Target Group
-   Set: Min: 1 Desired: 1 Max: 2

PHASE 6: CODEDEPLOY (BLUE/GREEN DEPLOYMENT)

1.  Create CodeDeploy Service Role

-   IAM → Create Role → CodeDeploy
-   Attach AWSCodeDeployRole

2.  Create CodeDeploy Application

-   Compute Platform: EC2/On-Prem

3.  Create Deployment Group

-   Deployment Type: Blue/Green
-   Select Auto Scaling Group
-   Attach Load Balancer
-   Enable automatic rollback

4.  Create S3 Bucket for CodeDeploy Artifacts Example:
    spotify-hybrid-codedeploy-artifacts

PHASE 7: APPSPEC FILE

Create appspec.yml:

version: 0.0 os: linux

files: - source: / destination: /home/ubuntu/app

hooks: AfterInstall: - location: deploy/scripts/install_dependencies.sh
timeout: 300 runas: ubuntu

ApplicationStart: - location: deploy/scripts/start_server.sh timeout:
300 runas: ubuntu

deploy/scripts/install_dependencies.sh:

#!/bin/bash docker pull

deploy/scripts/start_server.sh:

#!/bin/bash docker stop hybrid_recsys || true docker rm hybrid_recsys ||
true docker run -d –name hybrid_recsys -p 8000:8000

6.  Auto Scaling Group updates instances
